# AWS インフラ構築コスト試算(個人利用)

ap-study を AWS 上でホスティングした場合のコスト試算。**なるべく無料枠を使う**方針で、個人利用(自分+身近な数人)を前提にする。試算日: 2026-07-20。

---

## 0. 結論(TL;DR)

| 構成 | 月額の目安 | 補足 |
|---|---|---|
| **S3 + CloudFront(推奨・最安)** | **実質 0 円** | CloudFront の「常時無料枠(1TB/月)」で個人利用は完全に収まる。ドメインは `xxxx.cloudfront.net` を利用 |
| 上記 + 独自ドメイン | **約 80〜230 円/月** | Route 53 ホストゾーン $0.50/月 + ドメイン登録料(.com で年 $12〜15 ≒ 月 $1 前後) |
| AWS Amplify Hosting(一番簡単) | **0 円(12ヶ月)→ 以降ほぼ 0 円** | Git 連携で自動デプロイ。無料枠超過後も個人利用なら数十円/月程度 |
| フル AWS(同期も AWS 化) | **実質 0 円** | 同期を Supabase → DynamoDB + Lambda に置き換え。いずれも常時無料枠内 |

> **要点**: このアプリは**サーバー不要の完全な静的サイト**。個人利用の範囲なら AWS の常時無料枠だけで賄え、**独自ドメインを使わなければ月額はほぼ 0 円**で運用できる。

---

## 1. アプリ構成の分析(コスト算定の前提)

`npm run build` した実測値:

| 項目 | サイズ | 内訳 |
|---|---|---|
| **デプロイ総量(dist/)** | **約 13 MB** | 静的ファイルのみ |
| ├ 図表画像(figures/) | 約 11 MB | PNG 70 枚 |
| ├ JS/CSS(assets/) | 約 1.3 MB | JS は gzip で 377 KB |
| └ index.html | 1 KB | |

技術的な特徴(README / 要件書より):

- **Vite + React + TypeScript の SPA**。問題データ(640〜800問)は静的 JSON としてバンドルに同梱
- **バックエンドサーバー不要**。ランタイムで API を呼ばない
- 学習進捗は **localStorage**(ブラウザ内)に保存
- クラウド同期は**任意**で、現状は外部の **Supabase**(無料枠あり)を利用
- AI チャットは**任意**で、利用者自身の API キー(BYOK)かローカルブリッジ経由。**アプリ側にサーバーコストは発生しない**
- 利用者は「自分+身近な数人、一般公開しない」(要件書 1章)

→ **必要なのは「静的ファイル配信(+CDN)」だけ**。コンピュート(EC2/コンテナ)もマネージド DB も本質的に不要。

---

## 2. 想定トラフィック(個人利用)

コスト超過が起きないことを確認するための概算(かなり多めに見積もっても):

- 利用者: 10 人程度、1 人あたり毎日 30 分学習
- 図表を含め全データを何度も読み込む最悪ケースでも **月間データ転送量 ≒ 1〜5 GB**
  - 実際はブラウザ + CDN キャッシュが効くため、図表は初回のみ取得 → さらに小さくなる
- リクエスト数: 月間でも数万〜十数万リクエスト程度

**CloudFront 常時無料枠(月 1TB 転送 / 1,000万リクエスト)に対して桁違いに余裕がある。**

---

## 3. 構成案の詳細

### 案 A: S3 + CloudFront(推奨・最安)★

静的サイトホスティングの王道。S3 に置いて CloudFront(CDN)で配信する。

```
[ユーザー] → CloudFront (CDN/HTTPS/キャッシュ) → S3 (13MBの静的ファイル)
```

| サービス | 用途 | 個人利用でのコスト |
|---|---|---|
| **S3** | 静的ファイル置き場(13MB) | ストレージ 0.013GB × $0.023 ≒ **$0.0003/月(実質 0 円)** |
| **CloudFront** | CDN・HTTPS 配信 | 転送 1〜5GB・数万リクエスト → **常時無料枠(1TB/1,000万req)内で 0 円** |
| **ACM(SSL証明書)** | HTTPS 化 | CloudFront と併用で**無料** |
| 合計(cloudfront.net ドメイン利用時) | | **実質 0 円/月** |

- **メリット**: 最安・常時無料枠が手厚く**期限なしで 0 円が続く**。スケールにも強い
- **デメリット**: 初期設定(S3 バケット + CloudFront ディストリビューション + OAC)がやや手間。デプロイは `aws s3 sync dist/ && cloudfront create-invalidation` を手動 or CI 化
- **SPA 対応**: HashRouter を使っているため 404 リライト設定は不要(サブパスも `base: "./"` 済み)

### 案 B: AWS Amplify Hosting(セットアップが一番簡単)

GitHub リポジトリを繋ぐだけで `npm run build` → CDN 配信まで自動化される。

| 項目 | 無料枠(12ヶ月) | 本アプリの消費 |
|---|---|---|
| ビルド時間 | 1,000 分/月 | 1 ビルド約 1〜2 分 × 月数回 → 数分 |
| 配信データ転送 | 15 GB/月 | 1〜5 GB → 枠内 |
| ホスティング保存 | 5 GB/月 | 13MB → 枠内 |

- **無料枠超過後**(12ヶ月経過後): $0.15/GB 配信・$0.023/GB 保存・$0.01/ビルド分 → 個人利用なら**月数十円程度**
- **メリット**: Git push で自動デプロイ、HTTPS・独自ドメイン・プレビュー環境が数クリック
- **デメリット**: S3+CloudFront ほどの「期限なし 0 円」ではない(無料枠は 12ヶ月)。ただし個人利用の従量課金額は微少

### 案 C: フル AWS 化(同期も AWS に寄せる場合)

現状の同期は Supabase(外部・無料枠あり)。これを AWS だけで完結させたい場合:

```
[アプリ] → Lambda Function URL → DynamoDB (同期コード別のJSON 1行)
```

| サービス | 常時無料枠 | 本アプリの消費 |
|---|---|---|
| **DynamoDB** | 25GB 保存 + 25 RCU/25 WCU(≒2億リクエスト/月相当) | 数人分の小さな JSON → **0 円** |
| **Lambda** | 100万リクエスト + 40万 GB秒/月 | 同期時のみ起動 → **0 円** |
| **Lambda Function URL** | 追加課金なし(API Gateway 不要) | **0 円** |

- **メリット**: すべて AWS の**常時無料枠**内。外部依存(Supabase)が消える
- **デメリット**: Lambda 関数の実装が必要(現状 Supabase 直叩きの `src/lib` を書き換え)。**個人利用なら Supabase 無料枠のままでも 0 円**なので、無理に移行する必要は薄い

## 4. 独自ドメインを使う場合の追加コスト

`cloudfront.net` のままなら 0 円だが、独自ドメイン(例: `ap-study.example.com`)を使うなら:

| 項目 | コスト |
|---|---|
| Route 53 ホストゾーン | **$0.50/月**(無料枠なし) |
| Route 53 DNS クエリ | $0.40 / 100万クエリ(個人利用ではほぼ 0 円) |
| ドメイン登録料 | .com で **年 $12〜15**(≒ 月 $1 前後)。取得先は Route 53 でも他社レジストラでも可 |
| **追加合計** | **約 $1.5/月(≒ 200〜230 円/月)** |

> ドメインにこだわらなければこの費用は不要。まずは無料の `cloudfront.net` で始め、必要になってから足せばよい。

## 5. 【重要】2025年7月の AWS 無料枠改定について

2025-07-15 に AWS の無料枠制度が変わっている。**アカウント作成日で扱いが異なる**点に注意:

- **2026年の今、新規にアカウントを作る場合(新モデル)**:
  - サインアップで **$100 クレジット**、対象サービス利用で最大 **+$100**(合計最大 **$200**)。**有効期限 6ヶ月**
  - 「無料プラン」を選べば、クレジットを使い切る or 6ヶ月経過までは課金されない。以後は有料プランへ移行
  - **ただし「常時無料枠(Always Free)」の 30以上のサービスは新モデルでも引き続き利用可能** — CloudFront 1TB、DynamoDB 25GB、Lambda 100万req はここに含まれる
- **2025-07-15 より前に作成したアカウント(旧モデル)**:
  - 従来どおり 12ヶ月無料トライアル + 常時無料枠(S3 5GB 無料 など)

**本アプリへの影響**: 案 A(S3+CloudFront)は主に**常時無料枠**に依存するため、新旧どちらのアカウントでも、6ヶ月経過後でも **実質 0 円**が続く(S3 の 13MB 分 ≒ 月 $0.0003 は誤差)。

## 6. コストを 0 円に保つための実務チェックリスト

無料枠運用でも「設定ミスで予想外の請求」が起きないよう、以下を最初に設定しておく:

1. **AWS Budgets で予算アラートを作成**(例: 月 $1 を超えたらメール通知)— 新モデルでは無料プランで自動的に課金が止まるが、有料プラン移行後の保険として必須
2. **CloudFront の料金クラスを絞る**(`PriceClass_100` = 北米+欧州のみ、日本からのアクセスは日本のエッジになるが料金クラス次第。個人利用なら最安クラスで十分)。※日本のエッジを使いたい場合は `PriceClass_200` 以上
3. **S3 バケットは非公開**にし、CloudFront の **OAC(Origin Access Control)経由**でのみ読ませる(直リンクによる意図しない転送・公開を防ぐ)
4. **不要な S3 バージョニング / ログ**をオフにしておく(地味に容量課金の元になる)
5. 図表 PNG は将来的に **WebP 化**すると転送量をさらに削減できる(現状 11MB → 数MB に圧縮可能)。無料枠内なので必須ではない

---

## 7. 推奨

- **まずは 案 A(S3 + CloudFront)+ `cloudfront.net` ドメイン**で始めるのが最安・最も堅実。**月額実質 0 円**、期限なし
- 「Git push で自動デプロイしたい」を優先するなら **案 B(Amplify)** が手軽(無料枠 12ヶ月、以降も個人利用なら微少)
- **同期は当面 Supabase(無料枠)のままで問題なし**。AWS だけで完結させたくなったら 案 C で DynamoDB+Lambda に移行(これも常時無料枠内)
- **独自ドメインは任意**。必要になった時点で Route 53 を足す(+約 200円/月)
