# ap-study 要件ブリーフ

応用情報技術者試験(AP)の学習Webアプリ。2026-07-16 の壁打ちセッションで確定した要件。

## 1. 目的とターゲット

- 応用情報技術者試験の合格を目指す**初学者**向けの学習アプリ
- 利用者は自分+身近な人のみ(一般公開しない、認証なし)
- スマホのスキマ時間学習と、PCでの腰を据えた学習の両方に対応

## 2. スコープ

### 対象範囲
- **午前**(四択80問): 演習の主軸
- **午後**(記述式): 自己採点方式で対応
  - 収録分野: 問1 情報セキュリティ(必須)+ 文系セット(問2 経営戦略 / 問9 プロジェクトマネジメント / 問10 サービスマネジメント / 問11 システム監査)

### 収録データ
- IPA公開の過去問 **直近5年(10回)分**(令和3年秋〜令和8年春を想定。公開状況に合わせて調整)
- 午前: 約800問。**図表を含む問題も画像で収録**する
- 解説は開発時に事前生成して収録(ランタイムでのAI API呼び出しは不要)
- 各問題に出典を明記する(例: 「令和6年度秋期 応用情報技術者試験 午前 問32」)
  - IPAの過去問題は利用許諾申請不要・出典明記で利用可(著作権表記に従う)

## 3. 機能要件(優先順)

1. **分野別演習(午前)** — シラバス分類(大分類: テクノロジ/マネジメント/ストラテジ、中分類: セキュリティ・ネットワーク・DB等)から選んで演習
   - 開始時に問数を選択(5問 / 10問 / 20問)
   - **1問ごとに即時で正誤+解説を表示**(解説には「初学者ポイント」を含める)
2. **復習(間隔反復)** — 間違えた問題を簡易Leitner方式で再出題(翌日→3日→7日→14日、正解でボックス昇格)。「あとで復習」の手動追加も可
3. **弱点分析ダッシュボード** — 分野別正答率、弱点トピック一覧、学習量の推移。弱点分野からそのまま演習開始できる導線
4. **模試モード** — 本番同様80問・150分タイマー・合格ライン60%判定
5. **午後演習(自己採点)** — 問題文+設問表示 → 解答入力 → 模範解答・採点ポイント表示 → 自分で○△×を記録
   - スマホでも読めるようセクション折りたたみ・本文/設問間の行き来しやすさを作り込む

### 継続の仕掛け(控えめ)
- 連続学習日数、累計演習数の表示のみ(バッジ・レベル等はなし)

## 4. 画面構成

下タブナビゲーション(5タブ): **ホーム / 演習(午前) / 午後 / 分析 / 設定**

| 画面 | 内容 |
|---|---|
| ホーム | 試験日カウントダウン、今日の復習キュー、連続日数・累計、各モードへの導線 |
| 演習(午前) | 分野・問数選択 → 出題(進捗バー、出典表示)→ 即時判定+解説 |
| 午後 | 分野選択 → 長文読解(折りたたみ対応)→ 解答入力 → 模範解答比較 → 自己採点 |
| 分析 | 大分類/中分類の正答率バー、弱点トピック、演習導線 |
| 設定 | 試験日設定、同期コード、エクスポート/インポート |

## 5. 技術構成

- **スタック**: Vite + React + TypeScript(SPA)
- **ホスティング**: 静的ホスティング(GitHub Pages / Vercel)。URLを知る人のみ利用
- **問題データ**: 静的JSON+画像をビルドに同梱
- **進捗保存**: localStorage を一次保存とし、**クラウド同期**(Supabase想定)
  - **同期コード方式**: 初回に発行されるコード(UUID)を他端末に入力して同一データに紐付け。ログイン不要
  - 注記: 認証なしのため厳密なアクセス制御はない。保存対象が学習履歴のみでリスク小と判断
  - JSONエクスポート/インポートも用意(バックアップ・移行用)

### 進捗データモデル(概要)
- `attempts`: questionId、日時、正誤、モード(演習/復習/模試)
- `reviewQueue`: questionId、Leitnerボックス、次回出題日
- `pmSelfGrades`: 午後の設問ごとの○△×
- `settings`: 試験日、同期コード

## 6. データ整備パイプライン(本プロジェクト最大の工数)

1. IPA公式サイトから過去問PDF(問題冊子・解答例)をダウンロード
2. PDFから問題文・選択肢・正答を抽出してJSON化(図表は画像として切り出し)
3. Claude が全問の解説+初学者ポイントを生成し、分野タグを付与
4. 抽出結果・解説のレビュー(サンプリング検証)を経て収録

注意: 問題文はPDF抽出ベースとし、記憶からの書き起こしはしない(正確性担保のため)。

## 7. フェーズ計画

| Phase | 内容 |
|---|---|
| 0 | プロジェクト基盤(Vite+React+TS、ルーティング、デザイントークン、下タブ) |
| 1 | データパイプライン構築(まず1回分で抽出→JSON→解説生成を検証) |
| 2 | 午前演習MVP(分野別演習+即時解説+localStorage進捗) |
| 3 | 復習(間隔反復)+分析ダッシュボード |
| 4 | 模試モード |
| 5 | 午後演習(自己採点) |
| 6 | クラウド同期(Supabase・同期コード)+静的ホスティング公開 |
| 7 | データ拡充(10回分へ拡大) |

## 8. やらないこと

- ユーザー登録・ログイン(メール認証等)
- 午後のAI自動採点、オンデマンドAI解説(将来拡張の余地として構成上は妨げない)
- 全11分野の午後収録(文系セット以外)
- バッジ・レベル等の本格ゲーミフィケーション
